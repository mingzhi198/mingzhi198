1. Graphs

Graphs, which can reflect connections between nodes, are made up of nodes and edges. And the nodes connected directly to a node are called neghbors.

2. BFS(Breadth-first search)

BFS can help us solve two problems:

  1. Is there a path from one node to another?
  2. What is the shortest path between the two nodes.

If you have a graph like this, and want to find a person whose name ends with character ‘m’.

    1. Make a list of people to search.
    1. Go to each person in the list and check whether this person’s name ends with ‘m’.

    1. If none of your friends’ name ends with ‘m’, you have to see friends of your friends.
    1. When your search for someone from the list, add all of their friends to the list.

2.1 Get the shortest path

As we can see from the graph, your friends are first-degree connections, and friends of them are second-degree connections. We should always search the first-degree connections before searching the second-degree connections, and so on. Thus, we use queue as the data structure to store the list.

2.2 Queues

Compared with stack, which is LIFO(Last in, first out), queue is FIFO(First in, first out). It’s a very vivid picture, from which we can see this abstract concept.
