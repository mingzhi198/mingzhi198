In this chapter, I learned about recursion, which is very useful in many algorithms. Also, this chapter shows us how to break down a problem into a base case and a recursive case.

1. Recursion

If you have a box containing more boxes, in which there will be no or more boxes.
The key is in one of the boxes.
How can you do to search for the key?

1.1 Solution 1

  1. Make a pile of boxes to check.
  2. Pick a box and look through it.
  3. Add a box to the pile to look through later, if you find one.
  4. If you find the key, you’re done.
  5. Repeat.

1.2 Solution 2

  1. Look through a box.
  2. Go to step 1, if you find a box.
  3. If you find a key, you’re done.
  • Even though both solutions accomplish the same thing, the second one is clearer.
  • However, loops have better performance in reality.
  • Nevertheless, it’s essential to understand the concept since most significant algorithms use recursion.

2. Base case and recursive case

We have to tell the function when to stop recursing in a recursive function.
Thus, every recursive function has two parts: the base case, and the recurive case.

  • example
def countdown(i):
  print i
  if i <= 0:  # <------- Base case: we are done here.
    return 
  else:       # <----- Recursive case
    countdown(i-1)

3. Stack

Our computers also uses a stack internally named call stack. Here we use an example of factor to illustrate this concept:
5! = 5 * 4 * 3 * 2 * 1 ==> 5! = 5 * 4!
4! = 4 * 3!
3! = 3 * 2!
2! = 2 * 1!
1! = 1

def fact(x): 
  if x == 1:
    return 1 
  else: 
    return x * fact(x-1)

Summary

  • Recursive functions have two cases: the base case and the recursive case.
  • Stacks have two operations: push and pop.
  • All function calls go onto the call stack.
  • The call stack can take up lot of memory if it gets large.
